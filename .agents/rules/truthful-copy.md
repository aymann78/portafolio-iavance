## Copy truthfulness
Do not invent clients, metrics, case studies, integrations, team size, uptime, SLAs, pricing, or results.

## Positioning style
This site should sell capability, not fake proof.
Use language like:
- we build
- we can design
- we help reduce
- we create systems for
- we turn this process into this
Avoid language that implies a real delivered client case unless verified in repo or user instructions.

## Portfolio mode
This is a frontend-only portfolio with demos and concept builds.
If a project is not real client work, frame it as:
- interactive demo
- capability prototype
- concept build
- visual simulation
Never disguise concept work as real production deployment.
